# 钢琴练习辅助软件 (Android) - 项目需求文档 (PRD)

## 1. 项目概述
本项目是一款面向 Android 8.0 (API Level 26) 及以上平台的钢琴练习辅助应用。
**核心创新点**：放弃复杂的动态乐谱渲染，采用**“静态简谱图片 + JSON 坐标时间轴”**的方案。通过解析预先制作的 JSON 文件，在播放 **MP3**（MIDI 仅在服务端转码）时，于简谱图片上动态绘制高亮光标（逐小节精度），实现完美的视听同步与交互。

**内容与分发**：曲谱资源由**自建服务器**统一管理。服务器上部署 **Phase 0 打点小工具**（网页）与 **管理后台**（**用户名 + 密码**登录后上传谱面图、音频等）。**管理端若上传 MIDI，服务端须统一转码为 MP3** 后再发布，`data.json` 中的 `audioFile` 指向 MP3。Android **App 请求曲目列表与下载文件时携带固定 App Token**；拉取清单后按需下载完整乐谱包到本地沙盒，**离线播放**（练习时不依赖实时网络，除非用户主动同步或更新）。

## 2. 技术栈建议 (供 Cursor 参考)
*   **开发语言**: Kotlin
*   **UI 框架**: Jetpack Compose (全量使用，利用 `Canvas` 和 `Modifier.transformable` 实现图片缩放与高亮框绘制)。
*   **架构模式**: MVVM + Kotlin Coroutines + StateFlow。
*   **音频引擎**: 推荐使用 **ExoPlayer** (Media3)。它原生支持音频的变速 (Speed) 和变调 (Pitch) 功能 (`PlaybackParameters`)，且对时间进度的回调比原生 `MediaPlayer` 更精准。

## 3. 核心数据结构 (JSON Schema)
App 的核心驱动力是乐谱包，一个乐谱包包含：一张简谱图片、一个音频文件、一个配置文件 `data.json`。
`data.json` 的结构定义如下：
```json
{
  "songId": "santao_che_01",
  "title": "三套车",
  "imageFile": "sheet.jpg",
  "audioFile": "audio.mp3",
  "measures": [
    {
      "id": 1,
      "startTimeMs": 0,
      "endTimeMs": 2500,
      "rect": { "x": 50, "y": 120, "w": 200, "h": 80 }
    },
    {
      "id": 2,
      "startTimeMs": 2500,
      "endTimeMs": 4800,
      "rect": { "x": 260, "y": 120, "w": 180, "h": 80 }
    }
  ]
}
```
*(注：**产品约定**：下发给 App 的 `audioFile` **均为 MP3**（服务端对上传的 MIDI 等做转码）。坐标 `x, y, w, h` 基于图片原始分辨率的绝对像素值，App 端需按屏幕缩放做矩阵转换。)*

## 4. 核心功能需求 (Core Features)

### 4.1 乐谱渲染与高亮同步 (UI & Sync)
*   **图片展示**: 使用 Compose 渲染简谱图片，必须支持双指缩放 (Pinch-to-zoom) 和拖拽平移 (Pan)。
*   **动态高亮**: 开启一个协程，高频（如每 50ms）轮询 ExoPlayer 的 `currentPosition`。根据当前时间戳，在 `data.json` 中查找到对应的 `measure`，并在图片对应的 `rect` 坐标上使用 Canvas 绘制一个半透明的高亮遮罩（如黄色圆角矩形）。

### 4.2 交互控制 (Touch Controls)
*   **单击谱面 (Single Tap)**: 切换音频的 **播放/暂停** 状态。（控制栏的显示/隐藏**不得**占用单击谱面，见 `UI.md`：顶栏按钮、边缘区域或定时自动隐藏。）
*   **双击小节 (Double Tap)**: 
    *   获取用户双击在屏幕上的 `(x, y)` 坐标。
    *   将屏幕坐标逆向转换为图片的原始像素坐标。
    *   遍历 `data.json`，判断点击落在哪一个 `measure.rect` 内。
    *   如果命中，调用 `ExoPlayer.seekTo(measure.startTimeMs)`，实现进度跳转。

### 4.3 播放参数调节 (Playback Adjustments)
*   **速度调节 (Tempo)**: 提供 Slider，范围 0.5x 到 2.0x，调用 ExoPlayer 改变播放速度。
*   **音调调整 (Pitch)**: 提供 +/- 或选择器，按半音阶调整 **-12 到 +12**，调用 ExoPlayer 改变音调（需保证变速不变调，变调不变速）。

## 5. 辅助功能需求 (Auxiliary Features)

### 5.0 服务端（网页工具与管理后台）
*   **打点小工具 (Phase 0)**：部署在与管理后台同一台服务器（或同域路径下），用于在浏览器中对谱面图框选小节、绑定时间轴，导出/保存 **`data.json`**（结构见第 3 节）。
*   **管理页面**：**用户名 + 密码**登录（会话由服务端颁发，如 Cookie / JWT）；登录后可：
    *   上传**简谱/谱面图片**；
    *   上传**音频**（**MP3** 或 **MIDI**）；若上传 MIDI，**服务端自动转码为 MP3**（如 FFmpeg），发布目录与 `data.json` 中的 `audioFile` **仅引用 MP3 文件名**；
    *   上传或保存 **`data.json`**（可由打点小工具生成后上传，或后续扩展为管理页内嵌打点流程）。
*   **对外提供（App 端）**：**曲目列表**与**文件下载**接口须在请求头携带**固定 App Token**（由服务端配置、与管理员账号分离）；返回 JSON（含 `songId`、`title`、各文件路径、可选 `updatedAt`）及可下载的静态资源（图片、**MP3**、`data.json`）。**管理接口**仅允许已通过用户名密码认证的管理员访问。

### 5.1 曲库与同步 (Library & Sync)
*   **主路径：从服务器同步**：App 启动或用户下拉刷新时，使用配置的 **Base URL** 与 **App Token** 请求**曲目列表**（Token 写入 HTTP 头，见 `development.md`），展示可下载/已下载状态；用户选择曲目后，将 `data.json`、谱面图、**MP3** 等**下载到** `Context.filesDir/scores/{songId}/`，随后练习页**仅从本地文件**加载。
*   **删除**：左滑 **`SwipeToDismiss`** 删除**本机已下载副本**（不删除服务器上的资源，除非后续扩展「管理端删除」）。
*   **可选补充**：设置中配置 **Base URL**；**App Token** 为与服务端约定的固定密钥，建议 **BuildConfig / 本地安全配置** 注入，必要时在设置中提供覆盖项仅供调试。无网调试可保留「从本机 ZIP 导入」为**可选**。

### 5.2 A-B 段落复读 (A-B Loop)
*   **设点模式 (Repeat)**: 顶栏 **`Repeat` 按钮用于进入/退出「A-B 设点模式」**（不是一键开关循环本身）。进入设点模式后，再通过专用 UI（如「设为 A」「设为 B」、或点选小节等，实现阶段再细化）确定起点小节 A 与终点小节 B；设定完成并确认后，才进入**循环播放**状态。再次点击 `Repeat` 可退出设点模式或清除 A-B（具体以实现时交互文案为准）。
*   **循环逻辑**: 记录 A 的 `startTimeMs` 和 B 的 `endTimeMs`。当播放器进度到达 B 的结束时间时，自动 `seekTo` A 的开始时间。

## 6. 开发阶段划分 (AI 执行步骤)

*   **Phase 0: 服务端 — 打点小工具 + 管理后台 + 静态/API**
    *   *打点小工具*: 单页 `index.html`（或等价前端）：Canvas 加载图片、`<audio>` 播放；画框、绑定时间、导出 `data.json`；部署到服务器（HTTPS）。
    *   *管理页面*: **用户名密码**登录；上传谱面图、MP3/MIDI、`data.json`；**MIDI 上传后服务端转 MP3**（FFmpeg 等）；列表展示已发布曲目；后端按 `songId` 分目录落盘。
    *   *接口*: **App 专用**列表 + 文件下载，校验 **App Token**；**管理 API** 校验登录会话；建议 `updatedAt` 便于增量更新。
*   **Phase 1: Android 基础工程与曲库**
    *   *目标*: 搭建 Compose MVVM；实现**拉取曲目列表**、**下载乐谱包到沙盒**、本地目录读取与首页列表 UI；**可选**保留 ZIP 解压导入。
*   **Phase 2: 核心渲染与缩放**
    *   *目标*: 在练习界面加载图片和 JSON。实现图片的双指缩放、拖拽，并能根据 JSON 数据在图片上静态画出所有小节的测试边框（验证坐标系转换逻辑是否正确）。
*   **Phase 3: 音频引擎与同步高亮**
    *   *目标*: 接入 ExoPlayer。实现播放/暂停。实现时间轴轮询，让高亮框随着音乐在正确的小节上移动。
*   **Phase 4: 交互与高级控制**
    *   *目标*: 实现单击暂停、双击小节跳转进度。实现底部的速度 (Speed) 和音调 (Pitch) 调节面板。实现 A-B 循环逻辑。
